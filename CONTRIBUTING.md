# Contributing

Small, focused contributions are preferred.

## Scope

- Keep the repository lightweight.
- Prefer clear examples over new framework code.
- Do not add provider integrations or large optimization layers without a strong reason.

## Development

- Target Python 3.10+.
- Keep dependencies minimal.
- Add tests only where they improve confidence for the current scope.
- Prefer readable functions and small dataclasses over abstraction-heavy designs.

## Pull Requests

- Explain the problem being addressed.
- Keep changes narrow and easy to review.
- Update documentation or examples if behavior changes.
